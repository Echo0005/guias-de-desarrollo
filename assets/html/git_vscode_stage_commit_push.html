Git: Stage, Commit y Push en VS Code

Esta guia proporcionara una orientacion sobre el manejo de Git, en Visual Studio Code. Nos permitira tener un mayor control sobre que elementos iran a un commit y cuando queremos hacer un push (subir a Repositorio Remoto) con todos nuestros commit.

Stage:

Consiste en una área de preparación, todos los cambios (archivos modificados) son colocados en un Stage, antes de mandarlos a Commit. En pocas palabras, un Stage es una caja abierta donde vamos colocando los Cambios y siendo el Commit el momento de cerrar dicha caja para almacenarla.

Pasando modificaciones a Stage:

En Visual Studio Code contamos con el Control de código fuente:

Al realizar una modicacion de un archivo y guardarlo, veremos en el Control de código fuente, en la seccion Cambios, los archivos que podemos pasar a Stage. Debemos hacer click en el + :

Y ya dispondremos de los cambios en Stage:

Commit:

El Commit lo llevamos a cabo con el boton Confirmarción. Arriba de este boton tendremos un Input para colocar el Mensaje de nuestro Commit:

Push:

El Push sube todos los cambios al Repositorio Remoto.

Una vez que tengamos al menos un Commit, disponemos del boton Sincronizar cambios para realizar el Push :
