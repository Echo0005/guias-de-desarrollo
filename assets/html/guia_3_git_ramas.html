Branch (Rama): Teoria, Codigo y Trabajo

Teoria

Ramas o Versiones. Permiten tener varias versiones de un mismo proyecto al mismo tiempo.

Un Proyecto Git por default cuenta con una Rama Principal, llamada master o tambien main y es de donde partiran todas las demas Ramas que creemos.

En pocas palabras, una Rama parte de la Rama Principal (conteniendo todo lo que haya en Principal: archivos y commit's realizados) para convertirse en un Desarrollo Independiente de la Rama Principal. A partir de aquí, todo lo que hagamos en ambas ramas, ninguna tendra los cambios (add's, modify's, delete's) de la otra, seran desarrollos independientes.

Nos permiten tener un mayor control de "bloques de cambios", cuando decidimos incorporar a la Rama Principal y cuando retirarlos para revertir cambios. Mientras un Commit es un Cambio, una Rama es un conjunto de Commit's.

En un entorno de Equipo, las Ramas permiten separar el trabajo de cada uno de los integrantes, para luego ir incorporando cada Rama por separado a la Rama Principal. Permitiendo retirar una rama por el criterio que sea (algun bug, etc), continuar trabajando en la rama para luego reinsertar nuevamente nuestra Rama a la Rama Principal.

Sin las Ramas, tendriamos tan solo una Rama Principal con todo tipo de Commit's de muchas personas, dificultando la lectura del Historial y la comprobacion para encontrar un problema.

Un Proyecto en Git, con Ramas permite ser mas administrable cuando mas crece el proyecto y ayudando a dividir y evaluar mejor el trabajo de cada uno de los integrantes del equipo.

Codigo

  • Ver ramas: git branch
  • Crear rama: git branch nombre-de-rama
  • Renombrar rama: git branch -m nombre-antiguo-rama nombre-nuevo-rama
  • Seleccionar / Posicionarse en / Cambiar de rama: git checkout nombre-de-rama
  • Borrar rama: git branch -d nombre-de-rama (debemos estar situados/posicionados en otra rama que no sea la que vamos a borrar)
  • Ver diferencias entre ramas: git diff nombre-rama-1 nombre-rama-2
  • Fusionar ramas: git merge nombre-rama-a-fusionar nombre-rama-destino (debemos estar situados/posicionados en la rama destino)

Trabajo

Para comenzar a trabajar en una Rama, debemos Seleccionar o Posicionarnos en la rama.

Todas las modificaciones, archivos creados y eliminados, pasaran a ser de nuestra Rama una vez les hagamos Commit (mientras estamos posicionados en nuestra Rama).

Al volver a la Rama Principal podemos notar que los cambios que mandamos a Commit en la otra Rama nunca se produjeron en Rama Principal.

Se recomienda ser Concientes sobre en que Rama estamos trabajando para evitar problemas innecesarios (deshacer cambios, cambiar de rama, modificar lo que queriamos ahora si en la Rama esperada...).

Al momento de fusionar dos Ramas con un merge, debemos realizar un commit.
