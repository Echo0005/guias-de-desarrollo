CSS: Buenas Practicas

Las buenas practicas nos permiten un desarrollo mas organizado en cualquier ambiente. En CSS nos permitira tener un codigo mas: mantenible (a la hora de hacer modificaciones), legible (para nosotros y otras personas) y velocidad de trabajo.

Uso de Class

El uso de las clases en CSS es uno de los primeros cambios radicales. Nos puede costar al principio pero mas adelante nunca mas dejaremos de usarlas por su versatilidad y los problemas que nos puede ahorrar.

La abstraccion es la clave.

Para tener en cuenta:

Nunca dejar una Class asignada en el html la cual nunca ocupemos.

Evitar sobrecargar una Class con muchas Propiedades las cuales podriamos asignar a un Class mas especifico.

Si vemos que una Propiedad se repite en mas de una Class, debemos evaluar para crear una Class que lleve tal codigo que se repita. Esto nos permite escribir menos codigo CSS y usar una Class mas seguidamente en html. Un ejemplo recurrente puede ser la Propiedad CSS display: flex;, una clase llamada por ejemplo flex podria llevar tal propiedad y ya en el html asignamos varios Class a una misma etiqueta.

Uso de Archivos Externos

Utilizar archivos.css nos permite tener un html mas limpio y un css mas facil de ubicar. Si generamos un archivo.css separado para cada asunto, potenciamos mas sus beneficios ante la gestion.

Ordenar Propiedades CSS

Consiste en ordenar las propiedades CSS por orden alfabetico. 

                        
.ejemplo {
    border: 2px solid rgba(255, 255, 255, 0.288);
    border-radius: 1em;
    margin: 5em 0;
    padding: 0 1em 1.4em 1em;  
}

A la hora de leer propiedades, alfabeticamente las ubicaremos mas rapido.

Ordenar Class segun orden en HTML

Consiste en ordenar las clases en el archivo.css segun sean utilizados.
